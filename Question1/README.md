1. User Identity Verification

Commands used:
whoami
groups

Explanation:
These commands display my current username and the groups my user account belongs to.

2. Workspace Validation

Commands used:
pwd
ls -l

Explanation:
The pwd command shows the current working directory and ls -l lists files and directories in long format.

3. Environment Confirmation File

Commands used:
echo "Linux user environment verified" > user_info.txt

Explanation:
This command creates the user_info.txt file and writes the required confirmation message into it.

4. File Integrity Check

Commands used:
wc -c user_info.txt

Explanation:
This command shows the number of characters present in the user_info.txt file.

5. Learning the Tools

Commands used:
man mkdir

Explanation:
The system does not have the manual pages installed, so no manual entry was found. However, the -p option of mkdir allows creating parent directories automatically if they do not exist.

6. Home Directory Inspection

Commands used:
ls ~ | sort

Explanation:
This command lists the contents of the home directory in alphabetical order.

7. Log Investigation

Commands used:
grep "admin" log.txt

Explanation:
This command searches for the word "admin" in log.txt and displays only the matching lines.

8. System Information Check

Commands used:
uname -r

Explanation:
This command displays the Linux kernel version currently running.

9. Network Connectivity Test

Commands used:
ping -c 4 www.google.com

Explanation:
This command verifies network connectivity by sending ICMP packets to www.google.com.

10. System Health Awareness

Commands used:
uptime

Explanation:
This command shows system uptime, number of users, and load average.
